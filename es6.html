Notes

Arrow functions are always anonymous functions. They cannot be declared with a name like named functions can. e.g. function myFunction(){}
They can however be assigned to a variable. e.g. const bark = () => {alert("woof")}
As such if you are relying on detailed stack traces to named functions it may be a better idea to not use arrow functions.

If you need to access 'this' or the arguments param provided by javascript you should not use arrow functions.

Array Methods

Array.from

For things that are Array-like and you need to convert them to true arrays. E.g. When returning a group of things from document.queryselector this will return a nodelist you can use Array.from to wrap it and return an array. Array.from also accepts a second argument which is a map function that will allow you to change the data as it is being put in the new array.

Array.of

Simple function that you can pass any number of arguments to and it will create an array from all of them.

Object Literal updates

If variable name and value are the same e.g. const object = {name: name} you can just leave it as object = {name} and it wioll be implied. Also functions you can leave the function keyword out. e.g. object = {myFunction = function() {} } you can just do object = {myFunction(){}}
